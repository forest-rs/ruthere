# add visibility-aware change cursors

Allow incremental consumers to filter retained store changes through the same caller-supplied visibility policy used by snapshot and projection reads.

## Design

Retain enough metadata on expiry-driven removals to evaluate visibility after the entry is gone, then add visible cursor APIs over the existing change log. Keep policy caller-supplied and transport out of scope.

## Acceptance Criteria

Document the visibility-aware cursor boundary in a crate-local ADR; extend retained change payloads as needed for visibility filtering; add changes_since_visible and a cheap visibility-aware check; update the example to show member/public cursor views; add focused tests; keep fmt/clippy/test/doc/typos green.
