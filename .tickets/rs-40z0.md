# Reduce speculative runtime APIs and tighten retained-gap semantics

Remove public versioned view APIs and projection freshness/provenance additions from ruthere_store, keep retained-gap behavior, and make late-watcher recovery explicit rather than pretending a cursor reset performs a full resync.

## Design

ruthere_store owns reduction, visibility-filtered reads, and retained-log gap detection; it does not own cache invalidation semantics or automatic replay recovery. Roll back speculative versioned/freshness/provenance surface, keep explicit retained gaps, start new watcher cursors at the retained floor, and require callers to rebuild materialized state before manually advancing a stale cursor.

## Acceptance Criteria

Versioned/freshness/provenance public APIs and example usage are removed; retained-gap polling remains explicit; watcher/server docs and helpers no longer imply cursor movement alone performs resync; validations pass.
